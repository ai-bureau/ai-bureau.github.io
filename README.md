# AI Bureau

AI Bureau — двуязычный сайт бюро и контентная система, которая связывает
Notion, подготовку двуязычных авторских статей, Python-публикатор, GitHub и Hugo.

> Машини повинні працювати. Люди повинні думати.

## Текущее состояние

- Публичный сайт: <https://ai-bureau.github.io/>
- Репозиторий: <https://github.com/ai-bureau/ai-bureau.github.io>
- Основная ветка: `main`
- GitHub Pages: GitHub Actions workflow
- Основной язык сайта: украинский
- Второй язык: английский
- Поиск источников: вручную с помощью Codex
- Написание статей: Claude + Денис, готовые версии UA и EN
- Одобрение публикации: статус `Публікувати` в Notion
- Запуск публикатора: вручную, один проход за запуск
- Новый двуязычный pipeline реализован и проверен локально 14 июня 2026 года
- Для реального dry-run новой базы требуется подключить `AI Bureau Publisher`
  к базе `Публикации` в Notion

## Назначение проекта

AI Bureau помогает находить интеллектуальную рутину в рабочих процессах и
передавать её проверяемым AI-системам. Контент сайта объясняет эту позицию и
подтверждает её материалами авторитетных авторов.

Проект состоит из трёх частей:

1. **Источники.** Codex вручную ищет и проверяет материалы для тезисов и статей.
2. **Публикации.** Claude и Денис готовят полноценную статью сразу на UA и EN.
3. **Публикатор.** Python-скрипт проверяет и атомарно публикует обе версии.
4. **Сайт.** Hugo собирает двуязычный статический сайт, GitHub Actions
   публикует его на GitHub Pages.

## Общий поток

```text
Тезисы в Notion
    ↓
Codex вручную ищет и проверяет источники
    ↓
Claude + Денис создают UA + EN в базе Публикации
    ↓
Черновик → На review → Публікувати
    ↓
python -m publisher --env-file actual.env
    ↓
Один GitHub-коммит с двумя связанными Markdown-файлами
    ↓
GitHub Actions собирает Hugo
    ↓
https://ai-bureau.github.io/
    ↓
Статус публикации в Notion: Опубліковано
```

## Быстрый старт

### Проверить сайт локально

Из Windows PowerShell:

```powershell
.\start-site.bat
```

Открыть <http://localhost:1313/>. Остановка: `Ctrl+C`.

### Проверить готовые публикации без изменений

```powershell
python -m publisher --env-file actual.env --dry-run
```

### Опубликовать одобренные статьи

```powershell
python -m publisher --env-file actual.env
```

Или:

```powershell
.\start-publisher.bat
```

### Запустить тесты

```powershell
python -m pytest
```

## Ключевые гарантии публикатора

- Публикуются только записи со статусом `Публікувати`.
- Уже опубликованные записи не выбираются повторно.
- Notion обновляется только после успешного GitHub-коммита.
- Обе языковые версии создаются одним GitHub-коммитом.
- Повторный запуск распознаёт уже созданные идентичные файлы.
- Коллизия стабильного slug останавливает публикацию, а не меняет URL молча.
- Неизвестный Notion-блок останавливает публикацию, чтобы не потерять текст.
- Будущая `Дата публікації` создаёт файл сразу, но Hugo скрывает статью до даты.
- Невалидная карточка пропускается и не блокирует остальные.
- Ошибки записываются в ежедневный лог.
