# Как использовать AI Bureau

Это основная ежедневная инструкция владельца проекта. Технические подробности и
аварийные процедуры находятся в каталоге [`docs/`](docs/).

## Ежедневный рабочий процесс

### 1. Проверить готовые публикации

Открой базу `Публикации` в Notion.

Claude готовит обе языковые версии и переводит карточку в `На review`.
Перед разрешением публикации проверь:

- заголовки, описания и полный текст UA и EN;
- факты и ссылки на записи из базы `Источники`;
- отсутствие черновых заметок внутри страницы;
- одинаковый смысл обеих версий;
- переведи в `Публікувати`, если статью нужно отправить на сайт.

Статус `Публікувати` является явным разрешением на публикацию.

### 2. Проверить публикации без изменений

Из PowerShell в папке проекта:

```powershell
wsl.exe --cd . python3 -m publisher --env-file actual.env --dry-run
```

Ожидаемый результат при готовой статье:

```text
[DRY RUN] Название статьи -> content/uk/blog/article-slug.md, content/en/blog/article-slug.md
```

Если готовых статей нет:

```text
Нових публікацій немає
```

### 3. Опубликовать

```powershell
wsl.exe --cd . python3 -m publisher --env-file actual.env
```

Или двойным кликом/из PowerShell:

```powershell
.\start-publisher.bat
```

`start-publisher.bat` запускает настроенное Python-окружение через WSL. При
ошибке окно останется открытым и покажет exit code.

После успешной публикации:

1. В GitHub одним коммитом появляются UA и EN Markdown-файлы.
2. GitHub Actions запускает сборку.
3. В Notion статус становится `Опубліковано`.
4. Заполняется поле `Опубліковано`.

### 4. Проверить сайт

- Сайт: <https://ai-bureau.github.io/>
- Украинский блог: <https://ai-bureau.github.io/blog/>
- Английский блог: <https://ai-bureau.github.io/en/blog/>
- Actions: <https://github.com/ai-bureau/ai-bureau.github.io/actions>

Обычно deployment занимает меньше двух минут.

## Как подготовить карточку публикации

Обязательные поля:

| Поле | Что указать |
|---|---|
| `Назва` | Внутреннее рабочее название |
| `Заголовок UA`, `Заголовок EN` | Публичные заголовки |
| `Опис UA`, `Опис EN` | Короткие описания для сайта и SEO |
| `Slug` | Стабильный URL: латиница, цифры и дефисы |
| `Дата публікації` | Дата или время показа на сайте |
| `Статус` | `Публікувати` для разрешения публикации |

Рекомендуемые поля:

| Поле | Что указать |
|---|---|
| `Тезиси` | Связанные тезисы AI Bureau |
| `Источники` | Использованные материалы |

Внутри страницы должны находиться точные разделы `# UA` и `# EN`. Инструкция
для Claude находится на странице проекта в Notion.

## Отложенная публикация

Установи будущую `Дата публікації`, затем запусти публикатор обычным способом.

Оба файла будут сразу добавлены в GitHub, Notion получит статус `Опубліковано`, но
Hugo не покажет статью до указанного времени.

## Локальный просмотр сайта

Запуск из PowerShell:

```powershell
.\start-site.bat
```

Открыть:

```text
http://localhost:1313/
```

Локальный сервер работает через WSL. Терминал должен оставаться открытым. Для
остановки нажми `Ctrl+C`.

## Настройка на новом компьютере

Требования:

- Windows 10 или новее
- WSL с Bash
- Python 3.10 или новее
- Git

Создание Python-окружения:

```powershell
py -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -r requirements.txt
```

Для разработки и тестов:

```powershell
python -m pip install -r requirements-dev.txt
python -m pytest
```

Создай `actual.env` по примеру `.env.example`. Подробности:
[docs/SECURITY.md](docs/SECURITY.md).

## Логи

Логи публикатора:

```text
logs/publisher_YYYY-MM-DD.log
```

Основные сообщения:

- `[DRY RUN]` — выбран путь, но изменений нет;
- `[OK] published` — файл создан и Notion обновлён;
- `[OK] already exists` — существующий идентичный файл распознан, Notion
  обновлён;
- `[SKIP]` — карточка пропущена из-за неверных данных;
- `[ERROR]` — публикация статьи не завершена;
- `publication path already contains different content` — slug уже занят другим
  содержимым; исправь карточку, не меняя URL опубликованной статьи молча.

## Если публикация не сработала

1. Запусти dry-run.
2. Проверь обязательные поля карточки.
3. Открой сегодняшний файл в `logs/`.
4. Проверь GitHub Actions.
5. Не меняй вручную статус на `Опубліковано`, если файл не появился в GitHub.
6. Повтори обычный запуск после устранения причины.

Подробная диагностика: [docs/TROUBLESHOOTING.md](docs/TROUBLESHOOTING.md).

## Что не нужно делать

- Не добавляй `actual.env` или токены в Git.
- Не запускай публикатор повторно во время уже выполняющегося запуска.
- Не переименовывай поля и варианты статусов Notion без изменения кода.
- Не публикуй русскоязычные статьи.
- Не редактируй сгенерированный Markdown как основной способ исправления
  карточки. Источник правды до публикации — Notion.
