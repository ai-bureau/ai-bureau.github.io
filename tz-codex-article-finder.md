# ТЗ: Codex Article Finder — автоматический поиск статей для AI-бюро

## 1. Назначение

Скрипт автоматически ищет статьи-подтверждения для тезисов AI-бюро и заполняет таблицу Статьи в Notion. Решает задачу: контент-машина работает без участия Дениса — он видит только готовые карточки статей и ставит галочку «Публікувати».

---

## 2. Среда выполнения

- **Платформа:** Windows 10, ноутбук-сервер (работает 24/7)
- **Язык/стек:** Python 3.x, библиотеки: `notion-client`, `openai` (встроенный web search через OpenAI API)
- **Запуск:** автоматически по расписанию — cron через Task Scheduler Windows
- **Расписание:**
  - Первые 30 дней после запуска: 3 раза в неделю (пн / ср / пт)
  - Начиная с 31-го дня: 2 раза в неделю (вт / чт)
  - Дата первого запуска фиксируется в `config.env` при первом старте
- **Зависимости:**
  - Notion API Integration Token (уже настроен, хранится в `.env`)
  - OpenAI API Key (для web search, хранится в `.env`)
  - Доступ к интернету

---

## 3. Входные данные

- **Источник:** Notion, таблица «Тезисы»
  - Data source: `collection://06b828fc-8676-43d7-9d27-f015add2adc3`
- **Формат:** записи Notion с полями: Тезис (title), Блок (select), Статус пошуку (select)
- **Авторизация:** Notion Integration Token из `.env`
- **Валидация:** скрипт обрабатывает только записи со статусом «Не розпочато» или «В процесі». Записи со статусом «Знайдено» или «Не знайдено» — пропускает.

---

## 4. Алгоритм работы

### 4.1 Нормальный сценарий

1. Скрипт запускается по расписанию.
2. Подключается к Notion API, читает таблицу «Тезисы».
3. Фильтрует записи: статус «Не розпочато» или «В процесі».
4. Для каждого тезиса:
   a. Меняет статус тезиса на «В процесі».
   b. Формирует поисковый запрос: содержание тезиса + контекст блока.
   c. Запускает web search (OpenAI встроенный поиск).
   d. Из результатов отбирает до 10 кандидатов.
   e. Для каждого кандидата оценивает релевантность: статья должна **подтверждать** позицию бюро, а не просто касаться темы. Авторитетный автор с противоположной позицией — отклоняется.
   f. Фильтр языка: статьи на UA и EN — принимаются. Статьи на RU — отклоняются безусловно.
   g. Отбирает топ-5 по релевантности (если нашлось меньше 5 — берёт сколько есть).
   h. Для каждой отобранной статьи создаёт запись в таблице «Статьи» Notion:
      - Заголовок (title)
      - URL
      - Мова (UA / EN — определяется автоматически)
      - Статус: «Знайдена»
      - Дата додавання: текущая дата
      - Додав: «Codex»
      - Саммари: 2–3 предложения о чём статья и почему подтверждает тезис
      - Тезиси: relation → соответствующая запись в таблице «Тезисы»
   i. Меняет статус тезиса на «Знайдено».
5. Переходит к следующему тезису.
6. По завершении всех тезисов — записывает итог в лог-файл.

### 4.2 Нестандартные сценарии

- **Для тезиса не найдено ни одной подходящей статьи с первого запроса** → скрипт формирует второй поисковый запрос (перефразированный, более широкий). Если снова ничего — третий запрос (ещё шире, по ключевым словам блока). После трёх неудачных попыток: статус тезиса → «Не знайдено», в лог пишется причина (какие запросы были, сколько кандидатов отклонено и почему — по возможности).
- **Статья с таким URL уже есть в таблице «Статьи»** → дубликат пропускается, в лог пишется «дубль: [URL]».
- **Ошибка Notion API (сеть, rate limit, 5xx)** → пауза 60 секунд, повтор до 3 раз. Если не восстановилось — тезис остаётся со статусом «В процесі», в лог пишется ошибка. При следующем запуске скрипт подхватит его (статус «В процесі» входит в фильтр).
- **Ошибка OpenAI API** → аналогично: пауза 60 секунд, 3 попытки. При неудаче — тезис остаётся «В процесі», лог.
- **Тезисов со статусом «Не розпочато» / «В процесі» нет** → скрипт завершает работу, в лог пишет «Нет тезисов для обработки. Завершено.»

## 5. Выходные данные

- **Результат:** новые записи в таблице «Статьи» Notion
  - Data source: `collection://882c92e1-9948-42d6-9e7e-a67371630f9f`
- **Формат:** записи Notion с полями: Заголовок, URL, Мова, Статус, Дата додавання, Додав, Саммари, Тезиси
- **Если запись с таким URL уже существует:** пропустить (не дублировать)
- **Обновление тезисов:** статус тезиса меняется на «Знайдено» или «Не знайдено» после обработки

---

## 6. Обработка ошибок

- **Сетевая ошибка / таймаут** → пауза 60 сек, повтор × 3, затем лог + тезис остаётся «В процесі»
- **Rate limit Notion API** → пауза 60 сек, повтор × 3
- **Rate limit OpenAI API** → пауза 60 сек, повтор × 3
- **Статья на русском языке** → тихо отклоняется, в лог не пишется (штатная фильтрация)
- **Статья с противоположной позицией** → тихо отклоняется, учитывается в счётчике попыток
- **Непредвиденное исключение** → лог с полным traceback, скрипт завершается, не падает без следа

**Логирование:**
- Файл: `logs/article_finder_YYYY-MM-DD.log` (новый файл каждый день)
- Пишется: дата/время запуска, какой тезис обрабатывается, сколько статей найдено/отклонено, итог по каждому тезису, финальная сводка сессии
- Уведомление пользователя: только через лог-файл (Денис проверяет по желанию)

---

## 7. Ограничения

- Скрипт обрабатывает тезисы последовательно, не параллельно (чтобы не превысить rate limits API)
- Максимум 10 кандидатов на один поисковый запрос, в таблицу идут топ-5
- Язык поиска: UA и EN. RU — исключён безусловно, без исключений
- Дата первого запуска фиксируется в `config.env` автоматически при первом старте — на её основе скрипт определяет текущий режим расписания (3/неделю или 2/неделю). Изменить дату вручную можно отредактировав `config.env`
- Токены хранятся только в `.env`, не в коде
