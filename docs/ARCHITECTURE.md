# Архитектура

## Обзор

AI Bureau — статический сайт и управляемая через Notion контентная система.
Notion хранит тезисы и карточки статей. Python-публикатор преобразует только
одобренные карточки в Hugo Markdown и создаёт коммиты через GitHub Contents API.
GitHub Actions собирает и публикует сайт.

## Компоненты

### Notion

Две связанные базы:

- `Тезисы` — позиции AI Bureau, по которым ищутся подтверждающие материалы;
- `Статьи` — найденные материалы и их публикационный статус.

Notion является интерфейсом управления контентом и точкой человеческого
решения. Он не используется как транзакционная база сайта.

### Ручной поиск статей

Codex читает необработанные тезисы, выполняет веб-поиск, проверяет материалы и
создаёт карточки в Notion. Автоматический Article Finder намеренно не
реализован: содержательная оценка подтверждения тезиса требует критического
анализа.

### Python-публикатор

Точка входа:

```text
publisher/__main__.py
```

Основные модули:

| Модуль | Ответственность |
|---|---|
| `config.py` | Загрузка и валидация env |
| `notion_gateway.py` | Чтение карточек и обновление Notion |
| `github_gateway.py` | Чтение и создание файлов GitHub |
| `renderer.py` | Формирование Markdown и front matter |
| `slug.py` | Транслитерация и slug |
| `http.py` | Ограниченные повторы API-запросов |
| `service.py` | Оркестрация и порядок изменений |

Публикатор одноразовый: выполняет один проход и завершается. Он не является
сервисом и не требует NSSM.

### Hugo-сайт

- Hugo Extended `0.163.1`;
- PaperMod закреплён через Git submodule;
- собственная главная и шаблон статьи находятся в `layouts/`;
- фирменные стили находятся в `assets/css/extended/bureau.css`;
- украинский контент — `content/uk/`;
- английский контент — `content/en/`.

### GitHub Pages

Workflow `.github/workflows/hugo.yaml`:

1. checkout с submodules;
2. установка закреплённой версии Hugo;
3. production-сборка;
4. загрузка Pages artifact;
5. deployment.

## Поток публикации

```text
Notion query:
Статус = Публікувати
Опубліковано = пусто
        ↓
валидация обязательных полей
        ↓
чтение связанных тезисов
        ↓
детерминированный Markdown
        ↓
проверка GitHub path
        ↓
создание файла и commit
        ↓
PATCH Notion:
Статус = Опубліковано
Опубліковано = текущее время
        ↓
GitHub Actions → Hugo → Pages
```

## Порядок изменений и согласованность

GitHub изменяется раньше Notion. Это принципиально:

- если GitHub не принял файл, Notion остаётся `Публікувати`;
- если GitHub принял файл, но Notion временно не обновился, следующий запуск
  сравнит содержимое и завершит обновление без дубликата.

Это даёт практическую идемпотентность для повторного запуска.

## Формат статьи

Публикатор создаёт YAML front matter:

```yaml
title: Заголовок
description: Саммари
date: 2026-06-13
publishDate: 2026-06-15T10:00:00+03:00
sourceName: Заголовок
sourceURL: https://example.com/article
thesis: Связанные тезисы
draft: false
```

Тело содержит саммари AI Bureau. Шаблон `layouts/single.html` показывает
источник, позицию бюро и Telegram CTA.

## Отказоустойчивость

- Повторы: `429`, `500`, `502`, `503`, `504` и сетевые ошибки.
- Количество попыток и пауза задаются env.
- Ошибка одной статьи не блокирует остальные.
- Ненулевой exit code возвращается, если хотя бы одна публикация завершилась
  ошибкой.
- Ежедневные логи сохраняют traceback.

## Ограничения

- Публикатор не обновляет уже опубликованный Markdown.
- Нет автоматического перевода одной карточки в две языковые статьи.
- Нет автоматического удаления или отката публикации.
- Нет параллельной публикации.
- Нет постоянного процесса или встроенного расписания.
