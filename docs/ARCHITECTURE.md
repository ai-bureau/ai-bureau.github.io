# Архитектура

## Обзор

AI Bureau — двуязычный статический сайт и управляемая через Notion контентная
система. Notion разделяет исследовательские материалы и готовые публикации.
Публикатор не создаёт содержание: он проверяет готовые версии и публикует их.

## Компоненты

### Notion

Три связанные базы:

- `Тезисы` — позиции и идеи AI Bureau;
- `Источники` — проверенные Codex материалы и референсы;
- `Публикации` — готовые авторские статьи UA + EN.

Отдельная страница `Инструкция Claude: подготовка публикаций для сайта AI
Bureau` является рабочим контрактом для автора.

### Контентный процесс

```text
Тезисы Дениса
    ↓
Codex ищет и проверяет источники
    ↓
Claude + Денис создают статью
    ↓
точные готовые версии UA + EN в одной странице Notion
    ↓
Денис: Статус = Публікувати
```

### Python-публикатор

Публикатор выполняет один проход и завершается.

| Модуль | Ответственность |
|---|---|
| `config.py` | env и ID базы `Публикации` |
| `notion_gateway.py` | свойства, блоки страницы и обновление статуса |
| `notion_markdown.py` | безопасное преобразование поддерживаемых блоков |
| `renderer.py` | Hugo front matter и готовое тело |
| `github_gateway.py` | чтение файлов и атомарный Git commit |
| `service.py` | согласованность GitHub → Notion |

## Поток публикации

```text
Notion: Публікувати + Опубліковано пусто
    ↓
проверить поля, slug, # UA, # EN и языки
    ↓
преобразовать поддерживаемые Notion-блоки в Markdown
    ↓
проверить два стабильных GitHub path
    ↓
создать один commit с UA + EN через Git Data API
    ↓
Notion: Опубліковано + время
    ↓
GitHub Actions → Hugo → Pages
```

## Согласованность

- GitHub всегда изменяется раньше Notion.
- Обе версии создаются одним GitHub-коммитом.
- Если GitHub успешен, а Notion временно недоступен, повторный запуск распознаёт
  идентичные файлы и завершает обновление статуса.
- Если стабильный slug занят другим содержимым, публикация останавливается.
- Неизвестный Notion-блок останавливает публикацию вместо молчаливой потери.

## Формат файлов

Обе версии имеют общий `translationKey`:

```yaml
title: Public title
description: Search description
date: 2026-06-14
publishDate: 2026-06-14
translationKey: stable-slug
thesis: Related thesis
draft: false
```

После front matter находится полный готовый текст соответствующей версии.

## Ограничения

- Публикатор не пишет, не переводит и не редактирует содержание.
- Публикатор не обновляет уже опубликованный Markdown автоматически.
- Нет автоматического удаления или отката публикации.
- Поддерживается ограниченный набор Notion-блоков.
- Нет постоянного процесса или встроенного расписания.
