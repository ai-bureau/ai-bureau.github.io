## База `Тезисы`

| Поле | Тип | Допустимые значения / назначение |
|---|---|---|
| `Тезис` | Title | Формулировка позиции |
| `Блок` | Select | `Рутина і делегування`, `Як будувати AI-системи`, `Широкий контекст` |
| `Статус пошуку` | Select | `Не розпочато`, `В процесі`, `Знайдено`, `Не знайдено` |
| `Статті` | Relation | Связанные записи базы `Статьи` |

## База `Статьи`

| Поле | Тип | Обязательность для публикации |
|---|---|---|
| `Заголовок` | Title | Обязательно |
| `URL` | URL | Обязательно |
| `Мова` | Select: `UA`, `EN`, `RU` | Обязательно; публикатор принимает только `UA`/`EN` |
| `Саммари` | Rich text | Обязательно |
| `Тезиси` | Relation | Рекомендуется |
| `Дата додавання` | Date | Обязательно |
| `Дата публікації` | Date | Необязательно; fallback на `Дата додавання` |
| `Опубліковано` | Date | Заполняется публикатором |
| `Додав` | Select: `Codex`, `Claude`, `Денис` | Рекомендуется |
| `Статус` | Select | Управляет процессом |

Варианты `Статус`:

- `Знайдена`
- `На review`
- `Публікувати`
- `Опубліковано`
- `Відхилена`

## Запрос публикатора

Публикатор выбирает записи, удовлетворяющие одновременно:

```text
Статус = Публікувати
Опубліковано = пусто
```

## Обновление после публикации

После успешного GitHub-коммита:

```text
Статус = Опубліковано
Опубліковано = текущее время запуска
```

## Авторизации

Notion MCP и Notion Public API — разные подключения:

- MCP используется агентом для ручной работы с базами;
- Public API token из `actual.env` используется Python-публикатором.

Интеграции `AI Bureau Publisher` нужен доступ минимум к базам `Статьи` и
`Тезисы`. Без доступа к `Тезисы` публикация карточек с relation не сможет
прочитать заголовки тезисов.

## Изменение схемы

Переименование любого поля или варианта статуса требует изменения кода и
тестов. Не меняй схему только в UI Notion.
