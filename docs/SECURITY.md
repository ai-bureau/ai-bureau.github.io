# Безопасность

## Секреты

Реальные секреты находятся в `actual.env`, который исключён из Git.

Содержимое:

- `NOTION_TOKEN`
- `GITHUB_TOKEN`
- идентификаторы и эксплуатационные настройки

Запрещено:

- коммитить `actual.env` или `.env`;
- вставлять токены в документацию, issues, логи или ответы;
- передавать секреты через публичные каналы;
- печатать env целиком для диагностики.

## Минимальные разрешения

### Notion

Интеграции `AI Bureau Publisher` нужен доступ:

- читать базу `Публикации` и содержимое её страниц;
- читать связанные страницы `Тезисы`;
- обновлять свойства страниц `Публикации`.

Не предоставляй доступ ко всему workspace без необходимости.

### GitHub

Токену публикатора нужен доступ только к:

```text
ai-bureau/ai-bureau.github.io
```

Минимальное разрешение:

```text
Contents: Read and write
```

Публикатору не нужны административные разрешения, управление Actions или
удаление репозитория.

## MCP и Public API

Notion MCP авторизует AI-агента и не заменяет `NOTION_TOKEN` публикатора.
GitHub CLI авторизация также не обязана совпадать с `GITHUB_TOKEN` в env.

## Проверка перед commit

```bash
git status --short
git check-ignore -v actual.env .env logs public .tools
git grep -n -I -E '(secret_[A-Za-z0-9]|github_pat_|ghp_[A-Za-z0-9])'
```

Проверь результаты вручную. Значения в `.env.example` являются placeholders.

## Ротация токена

Если токен истёк или отозван:

1. создать новый токен с минимальными правами;
2. заменить значение только в `actual.env`;
3. выполнить dry-run;
4. отозвать старый токен;
5. проверить обычный запуск при наличии тестовой карточки.

## Действия при утечке

1. Немедленно отозвать токен у провайдера.
2. Создать новый токен.
3. Удалить секрет из текущих файлов.
4. Если секрет попал в Git, считать его раскрытым даже после удаления commit.
5. Проверить GitHub audit/activity и Notion activity.
6. Зафиксировать инцидент и причину.

## Публичность

Репозиторий и сайт публичны. Всё, что попадает в Git commit, считается
публичным навсегда.
