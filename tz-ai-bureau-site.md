# ТЗ: AI-Bureau — сайт + публикатор статей из Notion

Два связанных задания в одном документе:
- **Часть A** — разовая работа: построить сайт на Hugo и задеплоить на GitHub Pages
- **Часть B** — постоянная работа: скрипт-публикатор который следит за галочкой в Notion и публикует статьи на сайт

---

# ЧАСТЬ A: Сайт на Hugo

## 1. Назначение

Создать статический сайт AI-бюро на Hugo, задеплоить на GitHub Pages по адресу `https://ai-bureau.github.io`. Сайт двуязычный (UA + EN), адаптивный, с блогом для агрегированных статей и страницами позиционирования бюро.

---

## 2. Среда выполнения

- **Стек:** Hugo (extended edition, последняя стабильная версия)
- **Хостинг:** GitHub Pages, репозиторий `ai-bureau` под аккаунтом владельца
- **CI/CD:** GitHub Actions — автоматическая пересборка сайта при каждом коммите в `main`
- **Домен:** `ai-bureau.github.io` на старте (свой домен подключается позже через CNAME — предусмотреть место в конфиге)
- **Зависимости:**
  - GitHub аккаунт и Personal Access Token (для Actions и для публикатора из Части B)
  - Hugo extended edition (Codex устанавливает сам)

---

## 3. Структура сайта

### 3.1 Страницы

| Страница | URL (UA) | URL (EN) | Описание |
|----------|----------|----------|----------|
| Головна | `/` | `/en/` | Головний месседж + CTA |
| Для кого | `/dlya-koho/` | `/en/for-whom/` | Цільова аудиторія з живими прикладами |
| Послуги | `/posluhy/` | `/en/services/` | Що робимо, формати роботи |
| Блог | `/blog/` | `/en/blog/` | Агрегований контент по тезисах |
| Про нас | `/pro-nas/` | `/en/about/` | Філософія, бачення, команда |

### 3.2 Двуязычность

- Основная мова сайта — українська (`uk`)
- Вторичная — англійська (`en`)
- Перемикач мови у шапці на кожній сторінці
- Кожна стаття блогу існує в двох версіях: `content/blog/slug.uk.md` і `content/blog/slug.en.md`
- Якщо EN-версія відсутня — сторінка не показується в EN-розділі (не fallback на UA)

### 3.3 Головна страница — контент

**Заголовок:** «Машини повинні працювати. Люди повинні думати.» *(IBM, 1967)*

**Підзаголовок:** AI-бюро передає рутину машині — щоб людина могла думати, творити і жити.

**Три колонки** (кому це потрібно — коротко):
- Малий бізнес і підприємці
- Волонтери і НКО
- Фрілансери і експерти

**CTA-кнопка:** «Написати нам» → відкриває Telegram (посилання підставляється з конфігу)

### 3.4 Блог — структура поста

Кожна стаття блогу має:
- Заголовок
- Дата публікації (з відображенням на сторінці)
- Мова (UA / EN — визначається автоматично по файлу)
- Короткий опис (summary) — 2–3 речення, виводиться в списку блогу
- Повний текст — посилання на оригінал + саммарі від бюро
- Тезис який підтверджує (метадані, не відображається читачу)
- Коментарі: Disqus (один акаунт для обох мов)
- Telegram-кнопка під кожною статтею: «Обговорити в Telegram»

### 3.5 Відкладена публікація

- У frontmatter кожної статті є поле `publishDate` (дата + час, ISO 8601)
- Hugo не рендерить статтю поки `publishDate` не настав
- Публікатор (Частина B) заповнює це поле з Notion

## 4. Алгоритм розгортання (разовая работа Codex)

### 4.1 Нормальний сценарій

1. Codex створює GitHub репозиторій `ai-bureau` через GitHub CLI (`gh repo create`)
2. Ініціалізує Hugo-проект: `hugo new site ai-bureau`
3. Встановлює тему — мінімалістична, адаптивна, з підтримкою i18n. Рекомендована: **PaperMod** або **Ananke** (Codex вибирає ту що краще підтримує двомовність з коробки)
4. Налаштовує `hugo.toml`:
   - два мовних розділи (`[languages.uk]` і `[languages.en]`)
   - baseURL = `https://ai-bureau.github.io`
   - Disqus shortname = placeholder `ai-bureau-disqus` (замінити вручну після реєстрації)
   - Telegram посилання = placeholder (замінити вручну)
5. Створює всі сторінки зі структури 3.1 — з базовим контентом з Notion-сторінки проекту (див. розділ «Джерело контенту» нижче)
6. Налаштовує GitHub Actions: `.github/workflows/deploy.yml` — на кожен push в `main` запускає `hugo build` і деплоїть у гілку `gh-pages`
7. Вмикає GitHub Pages у налаштуваннях репозиторію (джерело: гілка `gh-pages`)
8. Робить перший коміт і пуш — сайт піднімається
9. Перевіряє що `https://ai-bureau.github.io` відкривається і обидві мови перемикаються

### 4.2 Джерело контенту для сторінок

Контент для статичних сторінок (Головна, Для кого, Послуги, Про нас) — брати з Notion-сторінки проекту:
- URL: `https://app.notion.com/p/37b3a8c895438112a693d0212facf339`
- Розділи: «Про проект», «Філософія бюро», «Кому це потрібно», «Що пропонуємо»
- Переклад на EN — Codex робить сам при створенні сторінок

### 4.3 Нестандартні сценарії

- **Тема не підтримує i18n з коробки** → Codex вибирає іншу тему яка підтримує, не патчить вручну
- **GitHub Pages не вмикається через API** → Codex виводить інструкцію: Settings → Pages → Source: gh-pages branch (одна ручна дія)
- **Помилка Hugo build** → Codex виправляє і повторює, не здає з помилками в CI

# ЧАСТИНА B: Публікатор статей (постійний скрипт)

## 1. Назначение

Скрипт слідкує за таблицею «Статьи» в Notion. Коли Денис ставить галочку ✅ «Публікувати» — скрипт генерує `.md` файл статті і комітить його в репозиторій `ai-bureau`. GitHub Actions автоматично пересобирає сайт.

Це єдина точка участі Дениса у всьому процесі.

---

## 2. Среда выполнения

- **Платформа:** Windows 10, ноутбук-сервер (24/7)
- **Язык:** Python 3.x
- **Запуск:** автоматично за розкладом через Task Scheduler Windows — кожні 2 години
- **Зависимости:**
  - `notion-client` — читати таблицю Статьи
  - `PyGithub` або `requests` — комітити файли через GitHub API
  - Notion Integration Token (`.env`)
  - GitHub Personal Access Token з правами `repo` (`.env`)

---

## 3. Входные данные

- **Джерело:** Notion, таблиця «Статьи»
  - Data source: `collection://882c92e1-9948-42d6-9e7e-a67371630f9f`
- **Фільтр:** статус = «Публікувати» AND поле «Опубліковано» = порожнє (щоб не публікувати двічі)
- **Поля які читаються:**

| Поле Notion | Використання |
|-------------|--------------|
| Заголовок | title у frontmatter |
| URL | посилання на оригінал в тілі статті |
| Мова (UA/EN) | визначає папку: `content/blog/` або `content/en/blog/` |
| Саммарі | description у frontmatter + перший абзац статті |
| Тезиси (relation) | метадані frontmatter, не відображаються |
| Дата додавання | якщо «Дата публікації» порожня — використовується як fallback |
| Дата публікації | поле `publishDate` у frontmatter (дата + час) |

---

## 4. Алгоритм работы

### 4.1 Нормальний сценарій

1. Скрипт запускається за розкладом.
2. Читає таблицю «Статьи» з Notion, фільтрує: статус = «Публікувати», поле «Опубліковано» порожнє.
3. Для кожної такої статті:
   a. Визначає мову → папку: UA → `content/blog/`, EN → `content/en/blog/`
   b. Генерує slug з заголовку (транслітерація для UA, lowercase-kebab для EN)
   c. Формує `.md` файл:

```markdown
---
title: "Заголовок статті"
date: 2026-06-12
publishDate: 2026-06-15T10:00:00+03:00
description: "Саммарі 2-3 речення"
thesis: "ID тезису з Notion"
draft: false
---

*Джерело: [Назва / Автор](URL)*

Саммарі від бюро: [текст саммарі з Notion]
```

   d. Комітить файл у репозиторій `ai-bureau` через GitHub API (гілка `main`)
   e. Оновлює запис у Notion: поле «Опубліковано» = поточна дата, статус → «Опубліковано»
   f. Пише в лог: `[OK] Запубліковано: {заголовок} → {шлях файлу}`
4. GitHub Actions автоматично пересобирає сайт після коміту.
5. Якщо нових статей немає — скрипт завершується з записом у лог: «Нових статей для публікації немає».

### 4.2 Нестандартні сценарії

- **`publishDate` у майбутньому** → файл комітиться зараз, але Hugo не показує статтю до настання дати. Це штатна поведінка — нічого додаткового не потрібно.
- **Slug вже існує в репозиторії** → до slug додається суфікс `-2`, `-3` і т.д. В лог пишеться попередження.
- **Поле «Дата публікації» порожнє в Notion** → використовується «Дата додавання» як `publishDate`. Стаття публікується одразу.
- **Помилка Notion API** → пауза 60 сек, повтор × 3. Якщо не відновилось — лог + запис у Notion не змінюється (спробує наступного разу).
- **Помилка GitHub API** → пауза 60 сек, повтор × 3. Якщо не відновилось — лог, статус у Notion НЕ змінюється (щоб наступний запуск спробував знову).
- **Мова статті не визначена** → скрипт пропускає статтю, пише в лог: `[SKIP] Не визначена мова: {заголовок}`.

## 5. Выходные данные

- `.md` файли в репозиторії `ai-bureau` у відповідних папках
- Оновлений статус у Notion: «Опубліковано» + дата
- Лог-файл: `logs/publisher_YYYY-MM-DD.log`

---

## 6. Обработка ошибок

- **Мережева помилка** → пауза 60 сек, повтор × 3, лог
- **Notion API rate limit** → пауза 60 сек, повтор × 3
- **GitHub API rate limit** → пауза 60 сек, повтор × 3
- **Непередбачений виняток** → лог з повним traceback, скрипт завершується без мовчазного падіння
- **Логування:** `logs/publisher_YYYY-MM-DD.log`, новий файл щодня

## 7. Ограничения и конфигурация

**Файл `.env` (мінімум):**
```
NOTION_TOKEN=...
GITHUB_TOKEN=...
GITHUB_REPO=ai-bureau/ai-bureau
GITHUB_BRANCH=main
NOTION_ARTICLES_DB=882c92e1-9948-42d6-9e7e-a67371630f9f
RETRY_PAUSE_SEC=60
MAX_RETRIES=3
```

**Поле «Опубліковано» в таблиці Notion:**
Якщо цього поля ще немає — Codex додає його через Notion API перед першим запуском. Тип: Date.

**Slug-генерація для UA:**
Використовувати транслітерацію (бібліотека `transliterate` або `unidecode`), lowercase, пробіли → дефіс, спецсимволи прибрати.
